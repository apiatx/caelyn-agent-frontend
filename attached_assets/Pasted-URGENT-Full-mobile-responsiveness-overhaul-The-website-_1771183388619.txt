4. Iframes & Widgets â€” Mobile Sizing
Iframes (TradingView, Finviz, CoinGecko embeds, etc.) need to be responsive:
css/* Make all iframes responsive */
iframe {
  width: 100% !important;
  max-width: 100%;
  height: auto;
  min-height: 300px;  /* Don't let them get too short */
  aspect-ratio: 16/9;  /* Maintain reasonable proportions */
}

/* On mobile, give iframes more vertical space since they're narrower */
@media (max-width: 767px) {
  iframe {
    min-height: 250px;
    aspect-ratio: 4/3;  /* Taller ratio on mobile */
  }
}
If any iframe has a hardcoded width or height attribute in the HTML tag, remove those and use CSS instead. Hardcoded pixel dimensions are the #1 cause of horizontal overflow on mobile.

7. Touch Targets
Every clickable element must be at least 44x44px on mobile (Apple's Human Interface Guidelines). This includes:

Nav links
Prompt buttons
Expand/collapse buttons on ticker cards
Close/minimize buttons
Filter chips
Any icon buttons

If any button or link is smaller than 44px, add padding to increase the touch target:

  }
}
8. Viewport Meta Tag
Make sure the HTML <head> includes:
html<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
The maximum-scale=1.0 and user-scalable=no prevent accidental pinch-zoom that breaks the layout. (Note: this is fine for a web app â€” NOT recommended for content sites, but for a trading dashboard it's the right call.)
